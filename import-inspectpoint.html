🔗 Import Inspect Point Reports

Not connected
Pulls inspection reports out of Inspect Point invoice emails (Martinez Fire). Each email's "Click here to view your invoice" link opens an Inspect Point page that has the Service Report PDF(s) attached — this tool follows that link, grabs the report (not the invoice), reads it to guess property / system / date, and on your OK uploads it to FLPS Software › FLPS Inspection Reports named EXT_… so it shows on the audit page. Nothing is uploaded until you click Import. Tip: click a row's subject to open the report PDF and verify before you set property/system/frequency/date. If one PDF covers two tests, click ⧉ to duplicate it and import under both systems. After importing, the email is tagged InspectPoint_Imported and the default search excludes that label.
ⓘ How it works & what it costs
How: The report PDF lives on Inspect Point, not in the email. The "view invoice" link and the PDF it hosts can't be fetched straight from the browser (cross-origin), so they're proxied through this app's own Cloudflare Worker (/api/fetch-report, locked to inspectpoint.com / cloudfront.net). The link is a public token — no Inspect Point login needed.

Matching: With Auto-read ON, Claude reads each report PDF and returns the property, system (sprinkler, FDC hydrostatic, internal pipe, backflow, …) and inspection date. The property is matched against your client list; the system + date drive the filename so the audit page slots it into the right row. Override any guess before importing. With Auto-read OFF, those fields come from the email only — free, but rougher.

Cost: Only Claude costs money (Google + Drive + the proxy are free). Each report is read once and cached; handled emails are excluded from later searches, so you don't re-pay. Rough per-report estimate by model:
